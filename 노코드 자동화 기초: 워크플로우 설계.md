### 비교 분석 

# Gmail 메일 자동 분류 및 Google Sheets 저장 자동화 비교

## 1. 실습 목표

| 항목 | 내용 |
|---|---|
| 목표 1 | 이메일 수신을 자동으로 감지한다. |
| 목표 2 | 조건에 따라 공지 메일과 일반 메일을 분류한다. |
| 목표 3 | 분류 결과를 Google Sheets에 저장한다. |
| 목표 4 | 자동화 흐름이 실제로 정상 동작하는지 확인한다. |

## 2. Make 자동화 구성

| 구분 | 내용 |
|---|---|
| 사용 도구 | Make |
| Trigger | Gmail Watch |
| Logic | Router를 사용하여 공지 메일과 일반 메일로 분기 |
| Action | Google Sheets의 2개 시트에 저장 |
| 저장 방식 | 공지 메일은 공지 시트와 전체 시트에 저장하고, 일반 메일은 전체 시트에 저장 |
| 특징 | 시나리오 흐름이 시각적으로 보이며, 복잡한 조건 분기와 데이터 가공에 유리함 |

## 3. Zapier 자동화 구성

| 구분 | 내용 |
|---|---|
| 사용 도구 | Zapier |
| Trigger | Gmail New Email |
| Logic | Paths를 사용하여 공지 메일과 일반 메일로 분기 |
| Action | Google Sheets의 2개 시트에 저장 |
| 저장 방식 | 공지 메일은 공지 시트와 전체 시트에 저장하고, 일반 메일은 전체 시트에 저장 |
| 특징 | 단계별 구성이 직관적이며, 빠르게 자동화를 만들기 쉬움 |
| 참고 사항 | Paths 기능은 요금제에 따라 사용이 제한될 수 있음 |

## 4. 자동화 흐름 비교

| 항목 | Make | Zapier |
|---|---|---|
| 이메일 감지 방식 | Gmail Watch 모듈 사용 | Gmail New Email Trigger 사용 |
| 조건 분기 방식 | Router 사용 | Paths 사용 |
| 저장 위치 | Google Sheets 2개 시트 | Google Sheets 2개 시트 |
| 공지 메일 처리 | 공지 시트와 전체 시트에 저장 | 공지 시트와 전체 시트에 저장 |
| 일반 메일 처리 | 전체 시트에 저장 | 전체 시트에 저장 |
| 실시간성 | 시나리오 실행 주기와 설정에 따라 달라질 수 있음 | 플랜과 polling 간격에 따라 달라질 수 있음 |
| 요금제 영향 | 작업량과 Operations 기준으로 영향 받을 수 있음 | Tasks 사용량과 Paths 기능 제한의 영향을 받을 수 있음 |

## 5. 검증 결과

| 검증 항목 | 결과 |
|---|---|
| Gmail 메일 수신 감지 | Make와 Zapier 모두 정상 작동함 |
| 공지 메일 분류 | 조건에 따라 공지 메일로 정상 분류됨 |
| 일반 메일 분류 | 조건에 맞지 않는 메일은 일반 메일로 정상 처리됨 |
| 공지 메일 저장 | 공지 시트와 전체 시트에 모두 저장됨 |
| 일반 메일 저장 | 전체 시트에만 저장됨 |
| Google Sheets 저장 | 두 도구 모두 지정한 시트에 정상 저장됨 |
| 전체 결과 | Make와 Zapier 모두 조건 분기와 시트 저장이 정상 작동함 |

## 6. Make와 Zapier 비교 분석

| 항목 | Make | Zapier |
|---|---|---|
| 기본 사용 난이도 | 처음에는 다소 복잡하지만 익숙해지면 유연하게 사용할 수 있음 | 인터페이스가 직관적이고 시작하기 쉬움 |
| 분기 처리 | Router를 통해 구조적으로 분기 가능 | Paths를 통해 간단하게 분기 가능 |
| 시각화 | 시나리오 흐름이 한눈에 보여 전체 구조 파악이 쉬움 | 단계별 구조가 단순하여 이해하기 쉬움 |
| 확장성 | 복잡한 자동화와 다양한 조건 처리에 강함 | 기본적인 자동화와 빠른 구축에 적합함 |
| 학습 곡선 | 중간에서 높음 | 낮음에서 중간 |
| 데이터 가공 | 변수, 함수, 필터 등을 활용한 세밀한 데이터 처리에 유리함 | 기본적인 데이터 매핑은 쉽지만 고급 가공에는 상대적으로 제한적일 수 있음 |
| 비용 구조 | Operations 사용량을 기준으로 영향을 받음 | Tasks 사용량을 기준으로 영향을 받음 |
| 고급 분기 기능 | Router를 활용해 비교적 자유롭게 구성 가능 | Paths 기능은 요금제에 따라 제한될 수 있음 |
| 실무 활용성 | 다양한 조건, 데이터 가공, 복잡한 업무 흐름에 유리함 | 빠른 자동화 구축과 단순 반복 업무 운영에 유리함 |

## 7. 장단점 비교 요약

| 항목 | Make | Zapier |
|---|---|---|
| 장점 1 | 복잡한 분기와 세밀한 제어가 가능함 | 인터페이스가 단순하고 사용하기 쉬움 |
| 장점 2 | 시나리오 흐름이 시각적으로 보여 구조 파악이 쉬움 | 자동화 제작 속도가 빠르고 입문자가 접근하기 쉬움 |
| 장점 3 | 데이터 가공, 필터, 조건 처리에 강함 | 다양한 앱 연동을 간단하게 설정할 수 있음 |
| 단점 1 | 처음 사용할 때 구조가 다소 복잡하게 느껴질 수 있음 | 복잡한 분기나 고급 자동화에는 상대적으로 제한적일 수 있음 |
| 단점 2 | 세밀한 설정이 필요한 경우 학습 시간이 필요함 | Paths 등 일부 고급 기능은 요금제 제한이 있을 수 있음 |
| 적합한 사용자 | 심화형 자동화, 확장성, 데이터 가공을 중시하는 사용자 | 입문자, 빠른 자동화 구현이 필요한 사용자 |

## 8. 결론

| 구분 | 내용 |
|---|---|
| Zapier | 직관적인 인터페이스를 제공하므로 빠르게 자동화를 만들고 싶은 경우에 적합함 |
| Make | Router 기반의 구조적 분기와 세밀한 제어가 가능하므로 복잡한 조건 분기와 확장성을 중시하는 경우에 적합함 |
| 전체 평가 | Zapier는 입문용 자동화에 강점이 있고, Make는 심화형 자동화와 복잡한 워크플로우 구성에 더 강점이 있음 |
| 실습 결론 | 두 도구 모두 Gmail 메일 감지, 조건 분기, Google Sheets 저장 기능이 정상적으로 작동함을 확인함 |

## 9. 추가 프로젝트 관점

| 항목 | 내용 |
|---|---|
| 프로젝트 | RSS 새 글 자동 수집 및 알림 시스템 |
| 사용 도구 | Make |
| 구성 예시 | RSS 피드 감지 → 새 글 데이터 수집 → 조건 필터링 → Google Sheets 저장 또는 알림 발송 |
| 확인한 점 | Make는 이메일 자동화뿐 아니라 RSS, 데이터 수집, 알림 시스템에도 활용 가능함 |
| 기대 효과 | 반복적인 정보 수집 업무를 자동화하여 시간 절약과 업무 효율 향상을 기대할 수 있음 |

## 10. 최종 요약

| 항목 | 요약 |
|---|---|
| 실습 내용 | Gmail 수신 메일을 자동 감지하고 조건에 따라 분류한 뒤 Google Sheets에 저장함 |
| Make 결과 | Router를 활용해 공지 메일과 일반 메일 분기가 정상 작동함 |
| Zapier 결과 | Paths를 활용해 공지 메일과 일반 메일 분기가 정상 작동함 |
| 공통 결과 | 공지 메일은 공지 시트와 전체 시트에 저장되고, 일반 메일은 전체 시트에 저장됨 |
| 비교 결과 | Zapier는 쉽고 빠른 자동화에 적합하며, Make는 복잡하고 확장성 있는 자동화에 적합함 |
| 최종 판단 | 두 도구 모두 실습 목적을 달성했으며, 사용 목적에 따라 적절한 도구를 선택하는 것이 중요함 |
